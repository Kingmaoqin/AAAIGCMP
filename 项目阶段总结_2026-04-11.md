# 项目阶段总结

## 1. 当前阶段的任务定义

### 1.1 当前任务到底是什么

当前项目已经从早期的“严格治疗效应估计（treatment effect estimation）”收缩并稳定为一个更明确的任务：

**两时间点高等级浆液性卵巢癌（HGSOC, High-Grade Serous Ovarian Cancer）CT 状态转移建模**。

更准确地说，当前系统包含三个彼此关联但层次不同的子任务：

1. **两时间点 CT 影像转移预测**
   - 给定病人在时间点 \(t_0\) 的 CT 影像、时间间隔信息和若干预先已知的弱临床/治疗元数据；
   - 预测时间点 \(t_1\) 的 CT 外观。

2. **关键变化区域（change region）代理预测**
   - 在没有可靠肿瘤真值分割标签的前提下；
   - 基于配准后影像差异，在一个受限的解剖有效区域内构造**变化区域代理标签（proxy change target）**；
   - 预测这些代理变化区域的位置与大小。

3. **两阶段的影像增强型阶段恶化预测（phase worsening forecasting）**
   - 不再直接宣称做“治疗效应估计”；
   - 而是将 pair 级响应定义为：从 pre scan 到 post scan 的治疗阶段是否恶化；
   - 基于冻结的图像主干特征与 pre-known metadata，预测 `response_phase_worsening`。

因此，当前任务同时具有以下属性：

- **时序建模问题**：建模 \(t_0 \rightarrow t_1\) 的状态转移；
- **图像预测问题**：预测下一时间点 CT；
- **代理分割/变化检测问题**：预测 proxy change region；
- **二分类问题**：在 stage-2 中预测 `response_phase_worsening`；
- **观测性弱治疗建模问题**：使用 observational treatment metadata 作为条件，但当前不做严格因果结论。

### 1.2 输入是什么，输出是什么

#### 子任务 A：影像转移预测

输入：

- 配准后的 pre-CT 体数据；
- 当前 slice 的 2.5D 邻域，即 \((z-1, z, z+1)\) 三张切片；
- anchor mask（解剖锚点掩膜）；
- 时间间隔和预知 metadata 条件向量。

输出：

- 对应 post-CT 中心切片的预测图像。

#### 子任务 B：关键变化区域代理预测

输入：

- 与子任务 A 相同；
- 训练时还使用由真实 pre/post 差分构造出来的 proxy change target。

输出：

- 当前 slice 的 `change_logits`；
- 经 sigmoid 和阈值化之后得到的 proxy change mask。

#### 子任务 C：两阶段响应预测

输入：

- 来自冻结图像主干的 pair-level imaging features；
- pre-known metadata 特征；
- 或者只使用 `phase_only` / `metadata_only` / `img_trans_only` 等特征子集。

输出：

- pair-level 的 `response_phase_worsening` 概率；
- 当前 `response_progression_like` 仅作 exploratory 观察，不作为主任务。

### 1.3 当前任务与最终大目标的关系

当前阶段**不是**最终的数字孪生系统，也**不是**严格的 treatment effect estimation 系统。

它是整个研究路线中的一个**中间支撑阶段**，目标是先把以下能力稳定下来：

1. 在真实卵巢癌多时间点 CT 上构建可复现的数据管线；
2. 做出可信的两时间点图像转移预测；
3. 在缺少肿瘤真值 mask 的前提下，至少建立 proxy 级别的变化建模能力；
4. 检验影像特征、阶段信息和 metadata 对“阶段恶化预测”是否有增量价值。

### 1.4 当前阶段解决的是整个系统中的哪一部分

如果把未来完整系统理解为“长期随访、影像更新、病灶变化、治疗响应、数字孪生更新”的整体框架，那么当前阶段解决的是其中的：

- **数据层**：多源卵巢癌 CT 队列构建；
- **预处理层**：解剖锚点引导的配准；
- **状态更新层**：两时间点影像转移模型；
- **弱响应层**：两阶段 phase worsening forecasting；
- **可视化层**：样本级图像与响应预测展示。

## 2. 当前模型总体结构

### 2.1 总体描述

当前系统是一个**多阶段（multi-stage）医学影像建模 pipeline**，其核心流程可概括为：

1. **多源数据整理与队列构建**；
2. **锚点分割与保守仿射配准**；
3. **2.5D 残差式图像转移模型训练**；
4. **从冻结图像模型提取 pair-level transition representation**；
5. **单独训练 response learner**；
6. **评估、分组统计与样图可视化**。

### 2.2 按模块划分的系统架构

| 模块 | 类型 | 主要职责 | 代码/文件位置 |
|---|---|---|---|
| 卵巢癌队列构建 | 数据处理模块 | 整理 CMB-OV 与 TCGA-OV，构建 session 与 pair 表 | `scripts/build_ovarian_ct_cohort.py`, `preprocessing/ovarian_cohort.py` |
| 锚点分割与配准 | 规则型/传统医学图像处理模块 | 提取骨盆/脊柱/大血管锚点，并做掩膜引导仿射配准 | `scripts/run_mask_guided_affine_registration.py`, `scripts/build_mask_guided_affine_dataset.py` |
| 图像转移模型 | 学习型模块 | 预测 post CT 和 proxy change logits | `scripts/train_ovarian_pair_ct.py` |
| 特征提取模块 | 学习型+数据导出模块 | 从冻结图像模型中提取 pair-level imaging features | `scripts/extract_ovarian_pair_features.py` |
| 响应学习模块 | 学习型模块 | 用 phase / metadata / imaging features 做 pair-level response 预测 | `scripts/train_ovarian_treatment_response.py` |
| 审计与稳健性评估 | 评估模块 | 做 LOPO、标签审计、pair roster 导出、系数分析 | `scripts/analyze_ovarian_treatment_response.py` |
| 可视化模块 | 可视化模块 | 渲染 pre/post/pred/change/probability 样图 | `visualization/render_treatment_response_samples.py` |

### 2.3 数据从输入到输出的完整流程

#### 训练流程

1. 从原始 `CMB-OV` 与 `TCGA-OV` 读取 session；
2. 按病人和时间排序，构造 `pre -> post` 相邻 pair；
3. 对 `CMB-OV` pair 做 `TotalSegmentator + SimpleITK affine` 配准；
4. 生成配准后的 NIfTI 体数据与锚点 mask；
5. 训练 stage-1 图像转移模型；
6. 用最优图像模型提取 pair-level 特征；
7. 在 stage-2 中训练 response learner。

#### 推理流程

1. 给定一个 pair 的 pre image 和 metadata；
2. 图像转移模型输出：
   - 预测 post 图像；
   - proxy change logits；
3. 冻结图像模型产生 pair-level transition features；
4. response learner 输出 `response_phase_worsening` 概率。

### 2.4 训练流程与推理流程的区别

两者的关键区别在于：

- **训练时**需要真实 post 图像来构造 proxy change target 和计算 loss；
- **推理时**没有真实 post 图像，模型只输出预测 post 和预测 change；
- **response 线**在当前版本中采用**two-stage** 设计，因此训练 response learner 时并不反向更新图像主干。

这意味着当前 response 模块不是和主图像模型端到端联合优化，而是**建立在主图像模型冻结特征之上的下游任务**。

## 3. 每一个板块分别用了什么模型、技术方法和原理

### 3.1 卵巢癌队列构建模块

#### 1. 模块目标

把来自 `CMB-OV` 和 `TCGA-OV` 的原始影像与 metadata 整理为可训练的：

- session 级表；
- 两时间点 pair 级表；
- 训练/验证/测试划分。

#### 2. 输入

- `CMB-OV` 原始 DICOM 目录与 portal metadata；
- `TCGA-OV` 原始影像目录与整理后的表格。

#### 3. 输出

- `ovarian_ct_sessions_master.csv`
- `ovarian_ct_two_timepoint_pairs.csv`
- `ovarian_ct_patient_splits.csv`
- 以及 CMB/TCGA 专用子表。

#### 4. 使用的方法

- 基于规则的 session 清洗；
- 病人级时间排序；
- 相邻时间点配对（adjacent pair construction）；
- patient-level split。

#### 5. 为什么选这个方法

因为当前任务明确收缩为“两时间点配对状态转移建模”，不再强求长序列 digital twin，因此：

- 相邻 pair 是当前最稳定、最可解释的样本单元；
- 也最适合小样本条件下的训练。

#### 6. 技术原理

对同一病人的时间点集合 \(\{t_1, t_2, \dots, t_n\}\) 按时间排序后，构造相邻 pair：

\[
(t_1 \rightarrow t_2), (t_2 \rightarrow t_3), \dots, (t_{n-1} \rightarrow t_n)
\]

这里：

- \(t_k\)：第 \(k\) 个时间点；
- \(t_k \rightarrow t_{k+1}\)：一个两时间点转移样本。

#### 7. 数学表达

设第 \(i\) 个病人的第 \(k\) 个 session 记为 \(s_{i,k}\)，则 pair 定义为：

\[
p_{i,k} = (s_{i,k}^{pre}, s_{i,k+1}^{post}, \Delta t_{i,k}, c_{i,k})
\]

其中：

- \(p_{i,k}\)：第 \(i\) 个病人的第 \(k\) 个 pair；
- \(s_{i,k}^{pre}\)：pre session；
- \(s_{i,k+1}^{post}\)：post session；
- \(\Delta t_{i,k}\)：两个时间点的天数间隔；
- \(c_{i,k}\)：该 pair 的 metadata 条件。

#### 8. 符号解释

- \(i\)：病人索引；
- \(k\)：时间点索引；
- \(p\)：pair；
- \(\Delta t\)：时间差。

#### 9. 训练时如何学习

该模块本身不学习，它是规则型数据构造模块。

#### 10. 推理时如何工作

推理时会读取某个已经构造好的 pair，作为模型输入样本。

#### 11. 当前限制

- `CMB-OV` 规模小；
- `TCGA-OV` 的治疗信息多为病人级，不是 visit-level action table；
- pair 的临床语义不完全一致。

### 3.2 锚点分割与掩膜引导仿射配准模块

#### 1. 模块目标

降低腹部 CT 在两时间点之间由于体位、呼吸、肠道充盈、腹水、扫描范围变化等带来的几何差异，使 pre/post 更可比。

#### 2. 输入

- pre CT；
- post CT；
- 通过 `TotalSegmentator` 自动得到的解剖锚点 mask。

#### 3. 输出

- 配准后的 moving 图像；
- 变换参数；
- 配准质量指标（如 MAE/PSNR 改善）。

#### 4. 使用的方法

- `TotalSegmentator` 自动分割稳定解剖结构；
- `SimpleITK` 进行 metric-mask-guided affine registration。

#### 5. 为什么选这个方法

腹部肿瘤本身变化大，若直接做自由形变配准，容易把真实病灶变化“拉平”或过拟合为形变；因此当前采用更保守的策略：

- 只对齐稳定骨性和血管结构；
- 只做 affine（平移、旋转、缩放、剪切）；
- 不做激进 non-rigid warping。

#### 6. 技术原理

先从 CT 中提取稳定锚点：

- 骨盆；
- 股骨；
- 腰椎/骶骨；
- 主动脉；
- 下腔静脉；
- 髂血管等。

再在这些锚点限定的区域上优化仿射变换 \(T\)：

\[
I_{move}^{warp}(x) = I_{move}(T(x))
\]

其中：

- \(I_{move}\)：moving image，即 pre image；
- \(I_{move}^{warp}\)：配准后的 pre image；
- \(x\)：空间坐标；
- \(T\)：仿射变换。

#### 7. 数学表达

仿射变换可写为：

\[
T(x) = A x + b
\]

其中：

- \(A\)：线性变换矩阵，编码旋转、缩放、剪切；
- \(b\)：平移向量。

优化目标是在固定掩膜区域 \(M\) 内最小化 fixed image 与 warped moving image 的差异：

\[
\min_{A,b} \; D\big(I_{fix}(x), I_{move}(A x + b)\big), \quad x \in M
\]

其中：

- \(I_{fix}\)：fixed image，即 post image；
- \(D(\cdot,\cdot)\)：图像相似性度量；
- \(M\)：anchor mask 所定义的区域。

#### 8. 符号解释

- \(I_{fix}\)：固定图像；
- \(I_{move}\)：待配准图像；
- \(A,b\)：仿射参数；
- \(M\)：配准度量掩膜。

#### 9. 训练时如何学习

该模块当前不是学习型模块，而是传统配准模块。

#### 10. 推理时如何工作

给定 pre/post pair，先运行 anchor segmentation，再运行仿射配准，输出配准后的 pre 供后续模型使用。

#### 11. 当前限制

- `TotalSegmentator` 给出的不是肿瘤 mask，而是解剖锚点；
- affine 只能处理全局几何差异，不能解决多灶病灶重分布；
- 对术后巨大解剖改变的处理能力有限。

### 3.3 图像转移模型（2.5D 残差式 CT 转移模型）

#### 1. 模块目标

学习给定 pre image 和条件向量时，如何预测 post image。

#### 2. 输入

对于某个中心层 \(z\)，输入为：

- \(x_{z-1}^{pre}\)：pre 的上一层切片；
- \(x_{z}^{pre}\)：pre 的中心切片；
- \(x_{z+1}^{pre}\)：pre 的下一层切片；
- \(a_z\)：anchor mask 当前层；
- \(c\)：metadata 条件向量。

#### 3. 输出

- \(\hat{x}_{z}^{post}\)：post 中心切片预测；
- \(g_z\)：change logits；
- （历史上还输出过 response logits，但当前不作为主线）。

#### 4. 使用的方法

- 轻量级 2D U-Net 风格 encoder-decoder；
- 在 bottleneck 处通过 FiLM 风格注入 metadata；
- 图像输出采用**残差预测**。

#### 5. 为什么选这个方法

原因有三：

1. 样本很少，直接上全 3D 大模型风险太大；
2. 腹部 CT 的相邻切片仍有明显局部连续性，2.5D 比单层更稳；
3. pre/post 之间存在明显“当前状态先验”，用残差预测比从零生成更合理。

#### 6. 技术原理

把当前中心切片作为 base image，网络不直接输出完整 post，而是输出一个残差图：

\[
\hat{x}_{z}^{post} = \mathrm{clip}\left(x_{z}^{pre} + r_\theta(s_z, c),\, 0,\, 1\right)
\]

其中：

- \(s_z = [x_{z-1}^{pre}, x_z^{pre}, x_{z+1}^{pre}, a_z]\)：2.5D 输入；
- \(r_\theta(\cdot)\)：由参数 \(\theta\) 的网络预测的残差；
- \(\mathrm{clip}\)：限制输出到归一化强度范围 \([0,1]\)。

#### 7. 数学表达

编码器-解码器可以抽象为：

\[
h_z = f_\theta(s_z, c)
\]

\[
r_z = d_\theta(h_z)
\]

\[
\hat{x}_{z}^{post} = \mathrm{clip}(x_z^{pre} + r_z, 0, 1)
\]

其中：

- \(h_z\)：瓶颈特征；
- \(f_\theta\)：编码与条件注入部分；
- \(d_\theta\)：解码与输出头；
- \(r_z\)：残差图。

#### 8. 符号解释

- \(x_z^{pre}\)：pre 中心切片；
- \(\hat{x}_z^{post}\)：预测的 post 切片；
- \(s_z\)：当前输入状态；
- \(c\)：条件向量；
- \(h_z\)：隐含特征；
- \(r_z\)：残差预测。

#### 9. 训练时如何学习

训练时最小化图像重建误差：

\[
\mathcal{L}_{img} = \| \hat{x}^{post} - x^{post} \|_1 + 0.5 \, \| \hat{x}^{post} - x^{post} \|_2^2
\]

其中：

- \(\|\cdot\|_1\)：L1 损失；
- \(\|\cdot\|_2^2\)：均方误差项；
- 系数 0.5 是手工设置的权重。

#### 10. 推理时如何工作

输入某个 pair 的 pre image 与条件向量，逐 slice 推理，输出预测的 post image。

#### 11. 当前限制

- 仍然是 slice-based，而不是完整 3D end-to-end；
- 预测结果是归一化图像域，不是原始 HU；
- 当前主要面向 CMB-OV 配准后数据，跨域泛化仍需观察。

### 3.4 变化区域代理监督模块

#### 1. 模块目标

在没有肿瘤真值 segmentation 的情况下，构造一个尽量与“病灶相关变化”接近的代理监督信号。

#### 2. 输入

- pre/post 图像；
- body mask；
- anchor mask；
- heuristic pelvic prior。

#### 3. 输出

- proxy change target；
- change valid region；
- 训练用的 change loss。

#### 4. 使用的方法

- 在 `non-anchor ∩ pelvic-prior` 区域内，对 `|post - pre|` 做阈值化；
- 再做形态学开闭操作和小连通域剔除。

#### 5. 为什么选这个方法

如果直接对全腹部做差分，会混入大量非肿瘤变化：

- 肠道气体；
- 膀胱充盈；
- 视野变化；
- 体位差异；
- 术后解剖扰动。

因此当前把监督区域限制在：

- 非锚点区域；
- 盆腔相关区域；
- 且只保留较稳定的残差连通域。

#### 6. 技术原理

定义：

- \(B\)：body mask；
- \(A\)：anchor mask；
- \(P\)：pelvic prior mask；
- \(V = B \cap (1-A) \cap P\)：valid region。

然后对残差幅度做阈值化：

\[
R = \mathbf{1}\left( |x^{post} - x^{pre}| \ge \tau \right)
\]

其中：

- \(\tau\)：差分阈值。

最终 proxy target：

\[
M^\* = \mathrm{Morph}(R \cap V)
\]

其中：

- \(M^\*\)：proxy change target；
- \(\mathrm{Morph}(\cdot)\)：形态学滤波，包括 opening / closing / 小连通域去除。

#### 7. 数学表达

变化头预测 logits \(g_\theta\)，经 sigmoid 得到概率：

\[
\hat{M} = \sigma(g_\theta)
\]

其中：

- \(\hat{M}\)：预测 change probability；
- \(\sigma(\cdot)\)：sigmoid 函数。

#### 8. 符号解释

- \(B\)：body mask；
- \(A\)：anchor mask；
- \(P\)：pelvic prior；
- \(V\)：有效监督区域；
- \(R\)：残差阈值图；
- \(M^\*\)：代理变化标签；
- \(\hat{M}\)：预测变化概率图。

#### 9. 训练时如何学习

当前使用联合 change loss：

\[
\mathcal{L}_{change} =
\lambda_{bce}\mathcal{L}_{bce}
\;+\;
\lambda_{tv}\mathcal{L}_{tversky}
\;+\;
\lambda_{area}\mathcal{L}_{area}
\]

其中：

- \(\mathcal{L}_{bce}\)：masked BCE-with-logits；
- \(\mathcal{L}_{tversky}\)：Tversky loss；
- \(\mathcal{L}_{area}\)：面积一致性损失；
- \(\lambda_{bce}, \lambda_{tv}, \lambda_{area}\)：对应权重。

当前一组重要配置为：

- `change_bce_weight = 0.35`
- `change_tversky_weight = 0.45`
- `change_area_weight = 0.20`
- `tversky_alpha = 0.7`
- `tversky_beta = 0.3`

#### 10. 推理时如何工作

先输出 `change_logits`，再做：

1. sigmoid；
2. 概率阈值化；
3. 形态学后处理；

得到最终 proxy change mask。

#### 11. 当前限制

- 这不是 lesion ground truth；
- `change_dice` 也不是 tumor Dice；
- proxy 仍可能混入腹水、软组织漂移和成像差异。

### 3.5 条件向量与特征融合模块

#### 1. 模块目标

把时间信息、弱治疗信息和 pre-known clinical context 注入图像模型，使图像转移预测受上下文约束。

#### 2. 输入

- 数值型 metadata；
- 类别型 metadata；
- 预定义的 one-hot 编码字段。

#### 3. 输出

- 一个固定长度的条件向量 \(c\)。

#### 4. 使用的方法

- 数值归一化；
- one-hot 编码；
- 在网络 bottleneck 使用 FiLM 风格条件注入。

#### 5. 为什么选这个方法

当前 metadata 多为小规模、异构、稀疏变量，最稳妥的方法是：

- 显式编码；
- 与图像主干进行 late fusion，而非过度复杂的 transformer 融合。

#### 6. 技术原理

在 bottleneck 特征 \(h\) 上生成尺度和平移参数：

\[
(\gamma, \beta) = \phi(c)
\]

\[
h' = h \odot (1 + \gamma) + \beta
\]

其中：

- \(\phi(\cdot)\)：条件网络；
- \(\gamma\)：缩放参数；
- \(\beta\)：平移参数；
- \(\odot\)：逐元素乘法。

#### 7. 符号解释

- \(c\)：metadata 条件向量；
- \(h\)：原始瓶颈特征；
- \(h'\)：条件调制后的特征；
- \(\gamma,\beta\)：FiLM 参数。

#### 8. 训练时如何学习

条件网络随主干一起训练，\(\gamma,\beta\) 通过反向传播自动优化。

#### 9. 推理时如何工作

根据当前 pair 的 metadata 构造 \(c\)，注入模型即可。

#### 10. 当前限制

- 治疗变量仍是粗粒度 observational metadata，不是完整 action table；
- 某些字段可能信息量很低，系数分析显示大量 metadata 在 stage-2 中接近 0。

### 3.6 两阶段特征提取模块

#### 1. 模块目标

从冻结的图像转移模型中抽取 pair-level 表征，用于后续 response 预测，而不反向污染图像主干。

#### 2. 输入

- 训练好的图像模型；
- pair 的所有有效 slice。

#### 3. 输出

- `ovarian_pair_features_stage2_v1.csv`
- 各种 imaging feature block。

#### 4. 使用的方法

- 从冻结主干的中间层做全局池化；
- 对每个 pair 在 slice 维度上做统计汇总。

#### 5. 为什么选这个方法

直接 joint multi-task 会显著伤害主图像任务，因此改用：

- 先学好 image transition；
- 再抽取 pair-level features；
- 最后单独做 response learner。

#### 6. 技术原理

对每个 pair 的所有 slice，提取若干特征：

- `img_pre_*`：较浅层 pre representation；
- `img_trans_*`：bottleneck transition representation；
- `img_pred_change_*`：预测 change 的统计摘要。

然后在 pair 内聚合，例如：

\[
z_{pair} = \mathrm{Agg}\left( \{ h_{z}^{(1)}, h_{z}^{(2)}, \dots, h_{z}^{(m)} \} \right)
\]

其中：

- \(z_{pair}\)：pair-level feature；
- \(h_z^{(j)}\)：第 \(j\) 个有效 slice 的特征；
- \(\mathrm{Agg}\)：聚合函数，如均值、标准差、最小值、最大值。

#### 7. 当前主要特征块及维度

| 特征块 | 维度 | 含义 |
|---|---:|---|
| `phase_only` | 5 | 只含阶段与时间信息 |
| `metadata_only` | 18 | 清理泄漏后的 metadata 特征 |
| `selected_metadata_only` | 10 | 更精简的 metadata 子集 |
| `img_pre_only` | 128 | pre 相关浅层影像表示 |
| `img_trans_only` | 256 | 最核心的 transition representation |
| `img_change_only` | 28 | 预测 change 的统计摘要 |
| `imaging_only` | 412 | 全部影像特征 |
| `imaging_plus_metadata` | 430 | 全部影像 + metadata |
| `img_trans_plus_selected_metadata` | 266 | 当前最值得关注的组合 |

#### 8. 训练时如何学习

该模块本身不训练，它只从已训练好的图像模型抽特征。

#### 9. 推理时如何工作

新 pair 到来时，可重复执行相同的 slice-level前向与聚合，得到 pair-level feature。

#### 10. 当前限制

- 特征维度相对样本数偏高；
- 全量 imaging features 在小样本下容易不稳定；
- 当前最有价值的是 `img_trans_only`，而不是 full imaging。

### 3.7 Stage-2 响应学习模块

#### 1. 模块目标

在不破坏主图像模型的前提下，预测 pair-level `response_phase_worsening`。

#### 2. 输入

- `phase_only`；
- `metadata_only`；
- `img_trans_only`；
- `img_trans_plus_selected_metadata`；
- 以及若干对照特征块。

#### 3. 输出

- \(p(y=1 \mid z)\)：pair-level 响应概率；
- 其中 \(y=1\) 表示阶段恶化。

#### 4. 使用的方法

- `LogisticRegression`；
- `SimpleImputer`；
- `StandardScaler`；
- optional `PCA`。

#### 5. 为什么选这个方法

因为当前样本极小（stage-2 仅 18 个 pair），如果上复杂 MLP 或树模型，容易过拟合且难解释。

Logistic regression 的优点是：

- 结构简单；
- 便于系数解释；
- 适合做信息源比较。

#### 6. 技术原理

对于特征向量 \(z\)，二分类 logistic 模型写为：

\[
p(y=1 \mid z) = \sigma(w^\top z + b)
\]

其中：

- \(z\)：输入特征；
- \(w\)：权重向量；
- \(b\)：偏置项；
- \(\sigma(\cdot)\)：sigmoid。

#### 7. 数学表达

logistic loss 可写为：

\[
\mathcal{L}_{resp} =
- y \log p
- (1-y)\log(1-p)
\]

其中：

- \(y \in \{0,1\}\)：真实标签；
- \(p = p(y=1 \mid z)\)：预测概率。

#### 8. 符号解释

- \(z\)：pair-level feature vector；
- \(w\)：模型权重；
- \(b\)：偏置；
- \(p\)：预测为恶化的概率；
- \(y\)：是否恶化。

#### 9. 训练时如何学习

当前 pipeline：

1. 读取 feature CSV；
2. 根据配置选择某个 feature block；
3. 在 train split 上做：
   - 缺失值填充；
   - `StandardScaler`；
   - 当维度过高时做 PCA；
4. 训练 `LogisticRegression(max_iter=2000, class_weight='balanced')`。

#### 10. 推理时如何工作

对新 pair 先提取 stage-2 特征，再经过同一套预处理和 logistic 模型，得到响应概率。

#### 11. 当前限制

- 当前主 endpoint `response_phase_worsening` 本质上更接近 **phase transition forecasting**；
- `response_progression_like` 在当前 split 上 train/val 单类，因此不可作为主监督任务；
- test 集只有 5 个 pair，且主要集中在 2 个病人上，单次 split 非常脆弱。

### 3.8 训练策略模块

#### 1. 模块目标

提高小样本条件下训练稳定性，并减少 domain mismatch。

#### 2. 主要策略

1. **TCGA-OV 预训练 \(\rightarrow\) CMB-OV 微调**
2. **不直接硬混合 TCGA 与 CMB 做主训练**
3. **change threshold calibration**
4. **group-balanced sampling（在某些分支中尝试）**
5. **response 采用 two-stage，而非 joint multi-task 主线**

#### 3. 原理与原因

直接把 `TCGA-OV + CMB-OV` 一起混训虽然病种同属 HGSOC 谱系，但两者数据生成机制不同，直接混合会伤害主任务。

因此当前最优策略是：

- 用 `TCGA-OV` 学域内表征；
- 再用 `CMB-OV` 做目标任务收口。

#### 4. 当前最优实践

- 主图像任务：`TCGA pretrain -> CMB finetune`；
- response 任务：固定图像主干 + stage-2 下游学习。

#### 5. 当前限制

- 训练样本依然非常少；
- balanced 分支对 change proxy 有帮助，但未必对图像质量最优；
- 没有大规模 repeated split 结果。

### 3.9 评估模块

#### 1. 模块目标

定量评估：

- 图像转移质量；
- proxy change 预测质量；
- pair-level response 预测质量。

#### 2. 使用的评估方式

- 图像指标：MAE、MSE、PSNR、SSIM；
- change 指标：`change_area_mae`、`change_area_ratio_mae`、`change_dice`；
- response 指标：AUC、ACC、F1；
- 稳健性评估：LOPO（Leave-One-Patient-Out）。

#### 3. 为什么需要 LOPO

因为 stage-2 的 test split 非常小：

- 18 个可用 pair；
- 8 个病人；
- 固定 test 只有 5 个 pair，且高度受个别病人主导。

因此单次 split 容易产生“幻觉性高分”，LOPO 更可信。

#### 4. 当前限制

- LOPO 仍然样本很少；
- AUC/F1 对超小样本很敏感；
- `response_progression_like` 由于 train 单类，当前不可学。

## 5. 每个数据集是如何清洗和处理的

### 5.1 CMB-OV 的清洗与处理

#### 原始问题

- 原始 DICOM 目录层级复杂；
- 视图中混有 scout / coronal / chest-only 等不合适 series；
- 时间点标签虽有，但仍较粗；
- 腹部前后时点几何差异大。

#### 清洗步骤

1. 读取 session 与 portal metadata；
2. 保留可用 CT series；
3. 过滤明显不适合的 series：
   - scout；
   - localizer；
   - coronal/sagittal；
   - 仅 chest 的 series；
4. 按病人和时间排序；
5. 构建相邻 pair；
6. 运行 anchor-guided affine 配准；
7. 导出 pair 级表。

#### train/val/test 划分

- 采用 patient-level split；
- 保证同一病人的不同 pair 不跨训练和测试。

#### 清洗后形成的数据表

核心结果包括：

- `ovarian_ct_cmb_sessions.csv`
- `ovarian_ct_cmb_longitudinal_pairs.csv`
- `ovarian_mask_guided_affine_pairs.csv`

### 5.2 TCGA-OV 的清洗与处理

#### 原始问题

- 原始用途更偏 baseline / radiogenomic；
- 治疗信息多为病人级历史；
- 影像协议和质量异质性更强；
- 不适合作为主 longitudinal response 数据。

#### 清洗步骤

1. 读取整理后的 session 表；
2. 选择可用 CT；
3. 按病人时间排序；
4. 构建 adjacent observational pairs；
5. 添加 protocol/geometry 风险字段；
6. 保留为辅助预训练资源。

#### 样本保留标准

- 有明确时间顺序；
- 影像可读；
- 不满足条件的 series 被过滤。

#### 清洗后形成的数据表

- `ovarian_ct_tcga_sessions.csv`
- `ovarian_ct_tcga_adjacent_observation_pairs.csv`
- `ovarian_ct_tcga_representation_sessions.csv`

### 5.3 配准与缓存构建

#### 处理步骤

1. DICOM / NIfTI 读入；
2. 重采样到统一尺寸（如 `64 x 64 x 32`）；
3. CT 强度裁剪并归一化；
4. anchor mask 同尺度缓存；
5. 生成配准后 moving volume；
6. 写入 NIfTI 与缓存文件。

#### 目的

- 统一模型输入空间；
- 降低 I/O 开销；
- 避免训练过程中重复读取和重复重采样。

### 5.4 Stage-2 数据处理

#### 原始问题

- 最初 metadata-only 结果异常偏高；
- 后来查明存在 leakage 字段：
  - `effect_proxy_label`
  - `pair_label_quality`
  - `history_treatment_label`

#### 当前处理方式

1. 删除上述泄漏风险字段；
2. 分出：
   - `phase_only`
   - `metadata_only`
   - `selected_metadata_only`
   - `img_trans_only`
   - `img_trans_plus_selected_metadata`
3. 对 train split 做：
   - `SimpleImputer`
   - `StandardScaler`
   - optional PCA

#### 当前结论

- 当前 clean stage-2 已经比旧版更可信；
- 但样本非常少，必须依赖 LOPO 而不是单次 split。

## 7. 之前尝试过什么方法

### 7.1 UCSF / LUMIERE / TaDiff 路线

#### 核心思路

- 早期曾尝试在脑胶质瘤 MRI 数据上做 treatment-aware diffusion 或类似任务。

#### 为什么当时值得尝试

- UCSF 数据有较明确的治疗历史；
- TaDiff-Net 本身是 treatment-aware longitudinal MRI 路线。

#### 为什么当前不再作为主线

- 病种不同，不是卵巢癌；
- 当前主任务已收缩为 HGSOC CT 两时间点转移建模；
- 因果/治疗效应结论在当前 ovarian 数据上更难成立。

#### 给出的启发

- “当前状态先验 + 残差更新”比从零生成更重要；
- 治疗信息必须先解决泄漏和语义问题。

### 7.2 无配准的 ovarian 相邻时间点 baseline

#### 核心思路

- 直接在原始 pre/post 对上做相邻时间点图像变化建模。

#### 问题

- 腹部位姿和器官位置差异太大；
- 原始差分混入大量非病灶变化。

#### 启发

- 配准是必要步骤；
- 但配准后仍需重新定义 change supervision。

### 7.3 学习型配准（VoxelMorph-style / TransMorph）

#### 核心思路

- 用 learned deformable registration 对齐腹部 CT。

#### 为什么尝试

- 医学图像配准中这些方法是常见学习型基线。

#### 为什么效果不够理想

- 当前队列太小；
- 腹部术后/治疗后形变复杂；
- learned registration 容易过拟合或引入不可信形变；
- 最终不如保守的锚点仿射配准稳定。

#### 启发

- 在当前数据条件下，传统保守配准比复杂 learned registration 更合适。

### 7.4 全腹 non-anchor residual 监督

#### 核心思路

- 在非锚点区域直接学习 change。

#### 问题

- 混入大量非肿瘤软组织变化；
- proxy supervision 太脏；
- Dice 难以稳定。

#### 启发

- 必须进一步加 pelvic prior；
- 必须做 morphology filtering。

### 7.5 直接混合 `TCGA-OV + CMB-OV` 共同训练

#### 核心思路

- 既然都是 HGSOC 谱系，就直接混在一起训练主模型。

#### 问题

- domain gap 明显；
- 虽然病种一致，但数据生成机制和标签语义不同；
- 直接混训会拉低主任务。

#### 实验现象

- 直接混训明显不如 `TCGA pretrain -> CMB finetune`。

#### 启发

- 当前更优策略是“先预训练，再微调”，而非同权混合。

### 7.6 joint multi-task response 分支

#### 核心思路

- 在 stage-1 图像模型中直接加 response head，共同训练。

#### 问题

- 反向梯度会伤害主图像任务；
- 响应标签极少，导致主任务被拖累。

#### 启发

- response 线必须拆成 two-stage。

### 7.7 response learner 泄漏字段版本

#### 核心问题

- 早期 `metadata_only` 用到了：
  - `effect_proxy_label`
  - `pair_label_quality`
  - `history_treatment_label`

#### 为什么这是严重问题

- 这些字段部分由 pre/post phase 共同派生；
- 会近似重编码标签。

#### 启发

- clean metadata 是必须的；
- 单次高分不可信，必须做 LOPO。

## 8. 当前的主要难点是什么

### 8.1 数据层面的难点

#### 难点

- `CMB-OV` 主任务样本太少；
- `TCGA-OV` 语义不完全一致。

#### 为什么难

- 模型很容易对少数病人过拟合；
- 任何一个 pair 的加入或移除都可能改变结论。

#### 已做缓解

- 使用 `TCGA pretrain -> CMB finetune`；
- 使用 LOPO。

#### 尚未解决

- 仍缺更大规模且更规范的 longitudinal ovarian CT 队列。

### 8.2 标签层面的难点

#### 难点

- 没有可靠 lesion ground-truth mask；
- `response_progression_like` 当前 train 单类。

#### 影响

- `change_dice` 只能是 proxy Dice；
- progression_like 无法作为主监督任务。

#### 已做缓解

- 使用 proxy change target；
- 明确把 progression_like 降级为 exploratory。

#### 尚未解决

- 真正 lesion-grounded response 仍无法验证。

### 8.3 配对与时序层面的难点

#### 难点

- 腹部影像时序差异复杂；
- pair 级 phase transition 分布不平衡。

#### 影响

- 某些 subgroup 非常难；
- response 本质可能被 phase 和 delta_days 主导。

#### 已做缓解

- patient-level split；
- LOPO；
- phase-only baseline。

#### 尚未解决

- 缺更细粒度、visit-level、可训练的 action-response 时序数据。

### 8.4 模型层面的难点

#### 难点

- 影像模型和 response 模型之间容易相互干扰；
- full imaging feature 在 stage-2 中不稳定。

#### 影响

- 多任务训练拖累主图像任务；
- feature 维度过高，容易过拟合。

#### 已做缓解

- 改为 two-stage；
- 聚焦 `img_trans_only`。

#### 尚未解决

- 还没证明影像增益在更大数据上稳定存在。

### 8.5 评估层面的难点

#### 难点

- test 样本极小；
- AUC/F1 对阈值与样本分布敏感。

#### 影响

- 单次 split 容易产生虚高结论。

#### 已做缓解

- 清除泄漏字段；
- 增加 LOPO；
- 对 response 单独审计。

#### 尚未解决

- 缺 repeated patient-level resampling 的均值和方差汇总。

### 8.6 临床意义层面的难点

#### 难点

- 当前响应标签更像阶段恶化，而不是严格治疗响应；
- 当前条件变量也多为弱治疗 metadata。

#### 影响

- 不能把结果写成强 treatment-effect 结论。

#### 已做缓解

- 任务重新命名为：
  `two-stage imaging-informed phase worsening forecasting under observational treatment metadata`

#### 尚未解决

- 仍需更严格、可追踪的治疗动作表和病灶标签。

## 10. 结果是怎么算出来的

### 10.1 MAE（Mean Absolute Error）

定义：

\[
\mathrm{MAE} = \frac{1}{N} \sum_{i=1}^{N} |\hat{x}_i - x_i|
\]

其中：

- \(N\)：像素或体素数；
- \(\hat{x}_i\)：预测值；
- \(x_i\)：真实值。

含义：

- 衡量平均绝对误差；
- 越小越好。

### 10.2 MSE（Mean Squared Error）

定义：

\[
\mathrm{MSE} = \frac{1}{N} \sum_{i=1}^{N} (\hat{x}_i - x_i)^2
\]

含义：

- 对大误差更敏感；
- 越小越好。

### 10.3 PSNR（Peak Signal-to-Noise Ratio）

定义：

\[
\mathrm{PSNR} = 20 \log_{10}\left(\frac{I_{max}}{\sqrt{\mathrm{MSE}}}\right)
\]

其中：

- \(I_{max}\)：图像动态范围上限，当前归一化后可取 1。

含义：

- 衡量重建质量；
- 越大越好。

### 10.4 SSIM（Structural Similarity Index）

SSIM 衡量亮度、对比度与结构相似性，常见定义为：

\[
\mathrm{SSIM}(x,\hat{x}) =
\frac{(2\mu_x\mu_{\hat{x}} + C_1)(2\sigma_{x\hat{x}} + C_2)}
{(\mu_x^2+\mu_{\hat{x}}^2+C_1)(\sigma_x^2+\sigma_{\hat{x}}^2+C_2)}
\]

其中：

- \(\mu_x, \mu_{\hat{x}}\)：均值；
- \(\sigma_x^2, \sigma_{\hat{x}}^2\)：方差；
- \(\sigma_{x\hat{x}}\)：协方差；
- \(C_1, C_2\)：稳定常数。

含义：

- 衡量结构相似性；
- 越大越好。

### 10.5 `change_area_mae`

定义：

先得到：

- 真实 proxy change 区域面积 \(A^\*\)；
- 预测 proxy change 区域面积 \(\hat{A}\)。

则：

\[
\mathrm{change\_area\_mae} = \frac{1}{M}\sum_{j=1}^{M} |\hat{A}_j - A^\*_j|
\]

其中：

- \(M\)：样本数；
- \(A_j^\*\)：第 \(j\) 个样本的真实 proxy 面积；
- \(\hat{A}_j\)：预测面积。

含义：

- 衡量预测变化区域大小是否合理；
- 越小越好。

### 10.6 `change_area_ratio_mae`

定义：

把变化面积除以有效区域面积 \(V\) 得到比例，再算 MAE：

\[
r_j^\* = \frac{A_j^\*}{|V_j|}, \qquad
\hat{r}_j = \frac{\hat{A}_j}{|V_j|}
\]

\[
\mathrm{change\_area\_ratio\_mae} = \frac{1}{M}\sum_{j=1}^{M} |\hat{r}_j - r_j^\*|
\]

含义：

- 降低不同切片有效区域大小不一致带来的影响；
- 越小越好。

### 10.7 `change_dice`

定义：

对预测 proxy change mask \(\hat{M}\) 与真实 proxy change mask \(M^\*\) 计算 Dice：

\[
\mathrm{Dice}(\hat{M}, M^\*) =
\frac{2|\hat{M}\cap M^\*| + \epsilon}{|\hat{M}| + |M^\*| + \epsilon}
\]

其中：

- \(|\cdot|\)：区域面积（或像素数）；
- \(\epsilon\)：防止分母为 0 的小常数。

含义：

- 衡量 proxy change 区域的重叠程度；
- 越大越好。

重要说明：

- 当前这里的 `Dice` 不是 lesion Dice；
- 它是 proxy change Dice。

### 10.8 Accuracy

定义：

\[
\mathrm{ACC} = \frac{TP + TN}{TP + TN + FP + FN}
\]

其中：

- \(TP\)：真正例；
- \(TN\)：真负例；
- \(FP\)：假正例；
- \(FN\)：假负例。

含义：

- 分类正确率；
- 但在类别不平衡时可能失真。

### 10.9 Precision / Recall / F1

定义：

\[
\mathrm{Precision} = \frac{TP}{TP+FP}
\]

\[
\mathrm{Recall} = \frac{TP}{TP+FN}
\]

\[
\mathrm{F1} = \frac{2 \cdot \mathrm{Precision}\cdot \mathrm{Recall}}
{\mathrm{Precision} + \mathrm{Recall}}
\]

含义：

- Precision：预测为正时有多少真的正；
- Recall：真实为正时抓住了多少；
- F1：二者平衡指标。

### 10.10 ROC-AUC

ROC-AUC 是 ROC 曲线下面积。其概率解释为：

随机抽取一个正样本和一个负样本时，模型给正样本更高分的概率。

记预测概率为 \(p_i\)，标签为 \(y_i\in\{0,1\}\)，则 AUC 越接近 1 越好，接近 0.5 说明接近随机。

### 10.11 当前项目里这些指标是如何计算的

#### 图像与 change 指标

在 `scripts/train_ovarian_pair_ct.py` 中按 slice 聚合计算。

#### response 指标

在 `scripts/train_ovarian_treatment_response.py` 中：

- clean split：在 validation 上扫描阈值，选择 F1 最优阈值；
- test 使用相同阈值；
- 输出 AUC、ACC、F1。

在 `scripts/analyze_ovarian_treatment_response.py` 中：

- LOPO 使用患者级留一评估；
- 当前汇总阈值默认取 0.5。

### 10.12 当前指标存在的问题

1. `change_dice` 是代理指标；
2. response 指标受极小样本影响明显；
3. `response_progression_like` 当前 train 单类，不能作为可靠指标；
4. 单次 split 的 AUC 可能虚高，因此 LOPO 更值得参考。

## 11. 当前阶段的结论

### 11.1 当前阶段已经完成了什么

1. 建立了可运行的 HGSOC 两时间点 CT 项目管线；
2. 完成了 `CMB-OV` 与 `TCGA-OV` 的统一数据整理；
3. 建立了 `TotalSegmentator + mask-guided affine` 的稳定配准方案；
4. 训练出了强性能的 two-timepoint image transition model；
5. 建立了 two-stage response learning 框架；
6. 识别并修复了 response 线中的 metadata leakage；
7. 建立了 LOPO 审计与样图可视化流程。

### 11.2 当前模型已经证明了什么

- `TCGA pretrain -> CMB finetune` 明显优于直接混训；
- 2.5D 残差式图像转移模型在当前数据上是可行的；
- 锚点引导仿射配准是有效的；
- `img_trans_only` 确实包含一定 response 相关信号；
- two-stage 设计比 joint multi-task 更合理。

### 11.3 还没有证明什么

- 还没有证明模型学到了真正的 lesion-grounded treatment response；
- 还没有证明当前结果可支持严格 treatment effect estimation；
- 还没有证明影像增益在更大样本上稳定超过 phase baseline。

### 11.4 哪些结论比较可靠

- 主图像任务已经可稳定工作；
- 当前 response 任务应被定位为：
  `two-stage imaging-informed phase worsening forecasting under observational treatment metadata`
- `response_progression_like` 当前不能作为主监督结论。

### 11.5 哪些结论仍然只是初步观察

- `img_trans_only` 是否真正带来稳健增量；
- 更复杂 metadata 是否有额外价值；
- 当前 proxy change 与真正病灶变化的一致程度。

### 11.6 当前工作在整个研究路线中的位置

当前工作是一个**方法收敛阶段**：

- 数据、任务与评估口径已经基本理顺；
- 现在已经可以进入更严谨的扩展和强化阶段；
- 但离最终临床级 treatment-response / digital twin 仍有距离。

## 12. 未来方向

### 12.1 数据层面

1. 补充更多 longitudinal ovarian CT；
2. 优先寻找或构建 lesion-level annotation；
3. 继续核查 TCGA-OV 是否存在可用的部分派生标注，但不能默认其可作为统一肿瘤真值。

### 12.2 特征层面

1. 保留 `img_trans_only` 为主影像特征块；
2. 对 stage-2 继续做小规模 selected metadata 组合；
3. 增加 protocol/coverage/surgery disruption flag 的系统分析，而不是简单过滤。

### 12.3 模型结构升级

1. 主图像模型短期内不建议盲目变大；
2. 更值得考虑：
   - 更好的 ROI-aware change head；
   - 更稳定的 pair-level transition aggregation；
   - 更针对阶段恶化的专门头。

### 12.4 时序建模加强

1. 当前是相邻 pair；
2. 中期可考虑把多个 pair 串成短轨迹；
3. 但前提是数据量和标签质量提升。

### 12.5 配对策略改进

1. 继续使用 adjacent pairs 作为主样本单元；
2. 对 phase transition 做更细分层；
3. 对 `baseline -> late_followup` 等困难子组做单独分析。

### 12.6 泛化能力增强

1. 继续保留 `TCGA -> CMB` 的预训练/微调模式；
2. 做 repeated patient-level resampling；
3. 明确 domain-aware 训练而非粗暴混合。

### 12.7 解释性增强

1. 保留 logistic regression 作为当前 response 主 learner；
2. 输出 feature coefficient 表；
3. 可视化 `phase_only` 与 `img_trans_only` 的预测差异。

### 12.8 如何更适合发论文

1. 论文叙事要诚实收口：
   - 主任务是 paired two-timepoint HGSOC CT transition modeling；
   - response 是 imaging-informed phase worsening forecasting；
   - 不是严格 treatment effect estimation。
2. 重点写清：
   - 数据异质性；
   - 配准设计；
   - proxy 监督逻辑；
   - 泄漏修复与 LOPO。

### 12.9 下一步最优先实验

1. 对 `phase_only / img_trans_only / img_trans_plus_selected_metadata` 做 repeated patient-level resampling；
2. 对最优图像模型做更多定性可视化和误差分析；
3. 抽取小规模人工 sanity-check 集，判断 proxy change 是否真的覆盖病灶相关变化。

### 12.10 中期与长期方向

#### 中期

- 找到更可信的 lesion proxy 或人工标注小集；
- 让 change task 更接近病灶变化。

#### 长期

- 发展成更完整的 ovarian digital twin update 模块；
- 在更可靠的治疗动作和病灶标签支持下，重新考虑真正的 treatment effect 建模。

## 13. 附录：术语与符号说明

### 13.1 缩写

- **HGSOC**：High-Grade Serous Ovarian Cancer，高等级浆液性卵巢癌
- **CT**：Computed Tomography，计算机断层扫描
- **MRI**：Magnetic Resonance Imaging，磁共振成像
- **LOPO**：Leave-One-Patient-Out，留一病人交叉验证
- **MAE**：Mean Absolute Error，平均绝对误差
- **MSE**：Mean Squared Error，均方误差
- **PSNR**：Peak Signal-to-Noise Ratio，峰值信噪比
- **SSIM**：Structural Similarity Index，结构相似性指标
- **AUC**：Area Under Curve，曲线下面积
- **ACC**：Accuracy，准确率
- **BCE**：Binary Cross Entropy，二元交叉熵
- **FiLM**：Feature-wise Linear Modulation，特征级线性调制
- **IDC**：Imaging Data Commons
- **SEG**：Segmentation Object
- **TIL**：Tumor-Infiltrating Lymphocyte，肿瘤浸润淋巴细胞

### 13.2 数学符号

- \(t_0, t_1\)：前后两个时间点
- \(\Delta t\)：时间间隔
- \(x^{pre}, x^{post}\)：pre/post 影像
- \(\hat{x}^{post}\)：预测的 post 影像
- \(z\)：slice 索引
- \(a_z\)：当前 slice 的 anchor mask
- \(c\)：metadata 条件向量
- \(h\)：隐含特征
- \(r_\theta\)：残差预测函数
- \(g_\theta\)：change logits 函数
- \(\sigma\)：sigmoid 函数
- \(B\)：body mask
- \(A\)：anchor mask
- \(P\)：pelvic prior
- \(V\)：有效 change 区域
- \(M^\*\)：proxy change target
- \(\hat{M}\)：预测的 change mask / probability
- \(w,b\)：logistic regression 参数
- \(y\)：二分类标签

### 13.3 关键数据字段名

- `pair_id`：pair 唯一标识
- `patient_id`：病人编号
- `delta_days`：前后时间点相差的天数
- `pre_treatment_phase_label`：pre scan 所处阶段标签
- `post_treatment_phase_label`：post scan 所处阶段标签
- `response_phase_worsening`：阶段是否恶化
- `response_progression_like`：是否进入 progression-like
- `history_treatment_code`：历史治疗背景编码
- `has_visit_level_treatment`：是否有 visit-level 治疗阶段信息
- `pre_has_chemotherapy`：pre 时点之前是否有化疗背景
- `pre_has_targeted_therapy`：pre 时点之前是否有靶向治疗背景
- `pre_has_any_radiation`：pre 时点之前是否有放疗背景
- `topographic_site_group`：解剖部位分组
- `histology_group`：组织学分组
- `pre_residual_disease_group`：残余病灶分组
- `pre_residual_tumor_group`：残余肿瘤分组
- `pair_label_quality`：pair 质量标签，当前不用于 clean response learner
- `effect_proxy_label`：pair 派生 effect 标签，当前不用于 clean response learner

### 13.4 模型简称

- **stage-1 图像模型**：2.5D 残差式 CT 转移模型
- **stage-2 response learner**：冻结图像主干特征上的 logistic regression
- **img_trans_only**：只用 transition representation 的影像特征方案
- **img_trans_plus_selected_metadata**：transition representation + 精简 metadata 方案

### 13.5 当前推荐的主实验口径

- 主图像任务：
  - `TCGA pretrain -> CMB finetune`
- 主 response endpoint：
  - `response_phase_worsening`
- response 主比较：
  - `phase_only`
  - `metadata_only`
  - `img_trans_only`
  - `img_trans_plus_selected_metadata`
- exploratory endpoint：
  - `response_progression_like`
